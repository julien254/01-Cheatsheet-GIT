1.Commande basique

  • $ git init :Initialiser un dossier .git dans le dossier courant.
  • $ git status :Voir les fichiers à ajouter ou à commit.
  • $ cat nom-du-ficher :Affiche le contenu du fichier nommer.
  • $ git stash : Enregistrer les fichiers sans les commit.
  • $ git stash pop : Recuperer les fichiers enregistrés.

2.Commit

  • $ git add nom-du-fichier :Ajouter le fichier dans l'index de git.
  • $ git commit -m "message pertinant d'écrivant les modification apporté au commit" : Commit tous les fichiers ajouter a l'index.-+

    • OU

  • $ git commit -am "message pertinant d'écrivant les modification apporté au commit" : Ajouter a l'index et commit les fichier qui ont deja etait rajouter une premiere foi.
  • $ git log : Afficher l'historique des commits.
  • $ git checkout sha-du-commit : Revenir au commit specifié.
  • $ git checkout master : Revenir au dernier commit.

3.Git avec github

/!\Important/!\

Avant de commit vos fichiers, il est trés important de "cacher" les fichiers sensible comme les mots de passe et les fichiers inutile pour ne pas qu'ils soit visible sur le repository.

Pour ce faire il suffit de creer un fichier .gitignore et d'y inscrire le nom des fichier que vous ne voulais pas commit.

  • $ git push origin nom-de-la-branche-principale : Envoyer tous les commits sur Github.
  • $ git pull origin nom-de-la-branche-principale : Récuperer les modifs sur Github.
  • $ git blame nom-du-ficher : Voir qui a mofier le fichier.
  • $ git show sha-du-commit : Connaitre les modifs en details du commit nommé.

4.Les Branches

  • $ git branch nom-de-la-branche : Creer une branche.
  • $ git checkout nom-de-la-branche : Changer de branche.

    • OU

  • $ git checkout -b nom-de-la-branche : Creer une branche et aller dessus.
  • $ git merge nom-de-la-branche : Fusionner la branche nommer avec la branche sur laquelle vous êtes.
